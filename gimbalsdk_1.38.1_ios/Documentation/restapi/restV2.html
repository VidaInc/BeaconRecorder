Places REST API

This document describes a Restful server API for interacting with the Gimbal V2 service.

API Endpoint https://manager.gimbal.com/api/v2/places

Authorization

Description

The Gimbal APIs require your organization's server API key in the AUTHORIZATION HTTP header. You can find your organization's server API Key under Organizations.

AUTHORIZATION: Token token=my_organization_server_api_key

Important Some Proximity APIs require OAuth access tokens. See below for more information about OAuth.

Content Type

Description

The Gimbal API's support JSON content type and should be specified in the Content-Type HTTP header as "application/json"

Content-Type: application/json

Resource Types

The following is the list of resource types used by the API.

Place

Attribute Data Type Description
id String Unique identifier for a Place
name String Name of the Place

Place Detail

Attribute Data Type Description Optional
id String Unique identifier for a Place false
name String Name of the Place false
attributes Object Series of key and value fields true
geofence Object Geofence object related to this Place true
beacons Array Array of Beacon objects related to this place true

Geofence

Attribute Data Type Description Optional
shape String CIRCLE or POLYGON false
radius Number The radius in meters of a Circle geofence true
center Point A Point object defining the center of a Circle geofence true
points Array Array of Points objects defining Polygonal geofences true
source String "1" for public geofence, null otherwise true
sourceId String public geofence id true
sourceVersion String Version for public geofence source true

Beacon

Attribute Data Type Description Optional
id String Unique identifier for the beacon false
name String Name of the beacon false

Point

Attribute Data Type Description Optional
latitude Number Latitude of this Point false
Longitude Number Longitude of this Point false

Response Status

Description

The following is the generic list of response status codes used by the API.

Response Status

HTTP Response Status Description
200 Success
401 Unauthorized
422 Unprocessable Entity
404 Resource Not Found
500 Unexpected Error

GET All Places

Description

Get all places for a specific organization.

HTTP Method

GET

URL

https://manager.gimbal.com/api/v2/places

Sample Request

curl -H "AUTHORIZATION: Token token=MY-GIMBAL-SERVER-API-KEY" https://manager.gimbal.com/api/v2/places

Sample Response

An array of Place Objects

[
    {
        "id": "MY-PLACE-ID-1",
        "name": "My First Place"
    },
    {
        "id": "MY-PLACE-ID-2",
        "name": "My Second Place"
    }
]

POST Place

Description

This call allows you to create a new Place for your organization.

HTTP Method

POST

URL

https://manager.gimbal.com/api/v2/places/MY-PLACE-ID

Sample Request

curl -X POST -H "AUTHORIZATION: Token token=MY-GIMBAL-SERVER-API-KEY" -H "Content-Type: application/json" -d '{ "name": "My Place", "geofence": { "shape": "POLYGON", "radius": null, "center": null, "points": [ { "latitude": 32.9, "longitude": -117.2 }, { "latitude": 32.9, "longitude": -117.2 }, { "latitude": 32.9, "longitude": -117.2 }, { "latitude": 32.9, "longitude": -117.2 } ], "source": null, "sourceId": null, "sourceVersion": null }, "beacons": [ { "id": "MY-BEACON-ID", "name": "My Beacon" } ], "attributes": { "key1": "value1", "key2": "value2" } }' https://manager.gimbal.com/api/v2/places

PUT Place

PUT request will replace all information with input submitted.

HTTP Method

PUT

URL

https://manager.gimbal.com/api/v2/places/MY-PLACE-ID

Sample Request

curl -X PUT -H "AUTHORIZATION: Token token=MY-GIMBAL-SERVER-API-KEY" -H "Content-Type: application/json" -d '{ "name": "My Place", "geofence": { "shape": "POLYGON", "radius": null, "center": null, "points": [ { "latitude": 32.9, "longitude": -117.2 }, { "latitude": 32.9, "longitude": -117.2 }, { "latitude": 32.9, "longitude": -117.2 }, { "latitude": 32.9, "longitude": -117.2 } ], "source": null, "sourceId": null, "sourceVersion": null }, "beacons": [ { "id": "MY-BEACON-ID", "name": "My Beacon" } ], "attributes": { "key1": "value1", "key2": "value2" } }' https://manager.gimbal.com/api/v2/places/MY-PLACE-ID

Sample Response

{
    "id": "MY-PLACE-ID",
    "name": "My Place",
    "geofence": {
        "shape": "POLYGON",
        "radius": null,
        "center": null,
        "points": [
            {
                "latitude": 32.9,
                "longitude": -117.2
            },
            {
                "latitude": 32.9,
                "longitude": -117.2
            },
            {
                "latitude": 32.9,
                "longitude": -117.2
            },
            {
                "latitude": 32.9,
                "longitude": -117.2
            }
        ],
        "source": null,
        "sourceId": null,
        "sourceVersion": null
    },
    "beacons": [
        {
            "id": "MY-BEACON-ID",
            "name": "My Beacon"
        }
    ],
    "attributes": {
        "key1": "value1",
        "key2": "value2"
    }
}

Notes

  • All existing attributes are deleted and replaced with input
  • All beacons are deleted and replaced with input
  • No existing beacons are affected, just the relationship to the place
  • Existing geofence is deleted and replaced with input
  • When updating a place with a geofence and beacon(s) using PUT, leaving either the geofence or beacon(s) blank will remove the geofence or beacon(s) from the place.

{"message":"Success"}
